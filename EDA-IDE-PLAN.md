## 1. Purpose & How to Use This Plan

### 1.1 Why this document exists
You are a senior VLSI engineer juggling SoC + PCIe + UCIe and (soon) many more IPs in parallel. Doom Emacs is your existing daily driver. Today the config is tuned for elfeed/org/Claude — but it does not yet behave like a real EDA IDE. This plan turns it into one **without throwing away anything that already works**. Every existing keybinding, every elfeed dashboard query, every life.org capture template, the slime/SBCL config, the eat paste fixes, the forge bindings — all preserved.

### 1.2 What "EDA IDE" means here
A single editor surface where you can, for any task in your portfolio:

1. **Read & write RTL** with tree-sitter syntax, LSP-driven navigation, instance hierarchy, beautify, formatter, lint, snippet templates.
2. **Run simulations** (Verilator / Icarus / cocotb) from `M-x` or `SPC c *` and jump to errors in compile mode.
3. **Run formal** (SymbiYosys + Yosys) and inspect traces.
4. **View waveforms** in GTKWave or Surfer launched from buffers.
5. **Drive git** (with worktrees + Forge over your internal Gitea) without leaving Emacs.
6. **Capture per-task notes** in a `project.org` next to the worktree, with all such files automatically appearing in `org-agenda`.
7. **Drive Claude Code** for agentic RTL/verification work, with per-IP `CLAUDE.md` and `.claude/agents/` context.
8. **Run up to 40 parallel tasks** without thrashing — using one Emacs daemon per IP family and one Doom workspace per task inside that daemon.

## 2. Full Q&A Log (decisions, rationale)

This is the canonical record of every architectural decision we already made together. **Do not relitigate.** When in doubt, this section wins.

### 2.1 Parallelism model — hybrid daemon + workspace + dynamic spawning

**Question.** With up to 40 parallel tasks across SoC, PCIe, UCIe (and more), do we use one giant Emacs, one Emacs per task, or something in between?

**Decision.** Hybrid. One Emacs daemon per **IP family** (initial seed: `soc`, `pcie`, `ucie`; user adds more on the fly). Inside each daemon, one Doom workspace (persp-mode perspective) per **individual task**. Plus a dynamic mechanism — `SPC k d n` from any Emacs frame and `eda new <name>` from the shell — to spawn a brand new IP-family daemon at runtime without restarting anything.

**Rationale.**
- A single Emacs with 40 perspectives runs out of memory and becomes painfully slow on GC.
- 40 separate Emacsen lose context sharing and waste startup energy.
- One daemon per IP family caps the blast radius: if PCIe Verilator process explodes a Yosys port, only the `pcie` daemon is affected. SoC and UCIe stay alive.
- Doom workspaces inside a daemon provide cheap per-task isolation (window config, buffer list, project root) without process overhead.
- The dynamic spawn mechanism is essential because IPs are added on the fly. We do not want to edit a launchd plist every time.

### 2.2 Claude integration mechanism — layered, never monolithic

**Question.** Claude Code CLI in vterm, `claude-code.el` Emacs UX, or `gptel` inline?

**Decision.** All three, in layers:

| Layer | Tool | Use case |
|-------|------|----------|
| L1 — agentic | Claude Code CLI in vterm/eat | Multi-step refactors, "fix all lint errors", autonomous build-test loops |
| L2 — Emacs-native | `claude-code.el` (stevemolitor) | Send buffer/region/error, transient menus, in-flow chat |
| L3 — inline | `gptel` with Anthropic backend | Inline single-shot completion in a code buffer, ad-hoc Q&A |

**Rationale.** Each layer has a sweet spot. CLI for autonomy, claude-code.el for "tell Claude about this exact buffer", gptel for "complete this single line." Per-IP `CLAUDE.md` and `.claude/agents/` files give every layer the same project context.

### 2.3 Local vs remote files — local on Mac, no TRAMP for primary work

**Question.** TRAMP to a Linux farm, or local on the Mac?

**Decision.** Local on the Mac filesystem under `~/eda/`. Push/pull to internal Gitea (`gitsrv.int.cspeed.io:3000`) for backup, code review, and CI. No TRAMP for primary editing.

**Rationale.** TRAMP latency multiplied by tree-sitter / LSP / flycheck / compile-mode reparses kills the daemon. Apple Silicon + 64 GB unified memory + local APFS is faster than any TRAMP path. CI on Linux farms is fine over git push.

### 2.4 EDA toolchain — open source only

**Question.** Synopsys, Cadence, Mentor (Siemens EDA)?

**Decision.** **None.** Open source only — Verilator, Icarus, cocotb, Verible, Yosys, SymbiYosys, GTKWave, Surfer, OpenLane2/OpenROAD (Docker on Mac). Commercial simulators are out of scope.

**Rationale.** Cost, licensing pain, Mac portability. PCIe Gen7 RTL is large but Verilator 5.x handles it. UVM works under Verilator+UVM-Python or via Verilator's UVM subset. For formal we use SymbiYosys + Yosys engines (smtbmc, btor, abc, aiger).

### 2.5 Daemon orchestration — hybrid + dynamic + persisted

**Question.** How exactly do daemons start, get attached to, and persist their state?

**Decision.**

- **Start.** `emacs --daemon=<name>` (or `--bg-daemon=<name>`). Seeded with `soc`, `pcie`, `ucie`. New ones spawned on demand by `SPC k d n` (from inside any running daemon) or `eda new <name> [path]` (from the shell).
- **Attach.** `emacsclient -s <name> -c` to attach a new GUI frame.
- **Persist.** Each daemon writes a per-daemon persp-mode state file at `~/.config/doom/.persp-state-<name>.el` on shutdown and loads it on startup. Daemon registry lives at `~/.config/doom/eda-daemons.el` (alist form, version-controlled).
- **List/kill.** `eda ls` shows running daemons (parsed from `pgrep -fa 'emacs.*--daemon='`). `eda kill <name>` calls `emacsclient -s <name> -e '(kill-emacs)'`.

**Rationale.** This isolates failures, persists workspace layouts per IP family, and works whether you boot from launchd, a shell, or another running Emacs.

### 2.6 Project layout — git worktrees, sibling to the bare repo

**Question.** One clone per repo per task, branches, or worktrees?

**Decision.** **Worktrees over bare-or-main checkouts.** Pattern:

```
~/eda/
├── pcie-controller/              # main checkout (or bare clone)
│   ├── .git/
│   └── rtl/, tb/, sim/, doc/
├── wt/
│   ├── pcie-controller-pcie7-link-init/   # worktree for one task
│   ├── pcie-controller-pcie7-ltssm-fix/   # another task on same repo
│   └── soc-top-clock-gen-rework/          # different repo
```

**Rationale.**
- Branch-switching in one directory is brittle when a long compile is mid-flight; worktrees give you an isolated `obj_dir/` per task.
- Worktrees share `.git/` objects so disk cost is small.
- magit's `magit-insert-worktrees` makes them first-class citizens in the status buffer.
- Doom workspaces map 1:1 to worktrees, giving you per-task buffer isolation.

### 2.7 SystemVerilog tooling — tree-sitter + Verible + verilog-ext

**Question.** verilog-mode (built-in), verilog-ts-mode, or verilog-ext?

**Decision.** All three, layered:

- `verilog-mode` — built into Emacs, foundational, AUTOs.
- `verilog-ts-mode` — tree-sitter major-mode for fast highlight/indent/imenu/which-func; uses `tree-sitter-systemverilog` grammar (full IEEE 1800-2023).
- `verilog-ext` — minor-mode that adds LSP wiring, hierarchy extraction, formatter, flycheck, navigation, snippets, project management.

**LSP server.** Primary: **Verible** (`verible-verilog-ls`). Secondary diagnostic: **slang** (`pip install pyslang` or build from source). Fallback for repos with quirky preprocessing: **svls** (`cargo install svls`).

**Rationale.** Verible has a Homebrew tap, is Apache-2 licensed, and provides ls + lint + format from one Apache project. Slang is the fastest/most-compliant frontend per chipsalliance test suite. svls is a Rust LSP useful when Verible chokes on `include path bombs.

### 2.8 Task knowledge model — per-task project.org + cross-IP org-roam

**Question.** Where does task knowledge live and how is it surfaced?

**Decision.** Two-tier:

- **Per task.** Every worktree gets a `project.org` file at its root. TODOs, debug log, links to RTL files, simulation results, formal results, references. Auto-discovered into `org-agenda-files` by a custom finder.
- **Cross-IP.** Org-roam nodes under `~/Dropbox/org-roam/<ip>/` (e.g. `pcie/`, `ucie/`, `soc/`). Permanent reference notes (protocol details, LTSSM state diagrams, sideband retimer notes, etc.). These outlive any single task.

The existing `~/Dropbox/organist-dinesh/life.org` Eisenhower-matrix capture is **untouched**. Project-task capture goes through a **new** template key (`p`, see 2.13) that writes to the current worktree's `project.org`.

### 2.9 IP families seed list — SoC / PCIe / UCIe, plus on-the-fly

Initial daemon seed (created on first run via `eda new`):

| Daemon name | Default root | Initial workspaces |
|-------------|--------------|---------------------|
| `soc` | `~/eda/wt/` filtered to `soc-*` | created lazily |
| `pcie` | `~/eda/wt/` filtered to `pcie-*` | created lazily |
| `ucie` | `~/eda/wt/` filtered to `ucie-*` | created lazily |

Adding a new IP family later is one command: `SPC k d n` -> prompt for name (e.g. `cxl`, `hbm-phy`, `dsp`) -> prompt for root -> spawn.

### 2.10 Tool install location — Mac only, Homebrew where possible

**Question.** Homebrew, source, Docker, Nix?

**Decision.**

| Tool | Method | Reason |
|------|--------|--------|
| Verible | `brew tap chipsalliance/verible && brew install verible` | Official tap |
| Verilator 5.x | `brew install verilator` | Core formula, arm64 bottle |
| Icarus Verilog | `brew install icarus-verilog` | Core formula, arm64 bottle |
| Yosys | `brew install yosys` | Core formula, arm64 bottle |

| Surfer | `cargo install --git https://gitlab.com/surfer-project/surfer surfer` | No brew formula yet, Rust |
| OSS CAD Suite (sby, GHDL, Bitwuzla, Z3, …) | tar.gz from YosysHQ releases, unpack to `~/oss-cad-suite/` | Bundles sby, Z3, Boolector, Bitwuzla, ABC, GTKWave, Surfer for darwin-arm64 |

| OpenROAD / OpenLane2 | Docker (or Nix flake) | Native macOS build is unsupported |
| direnv | `brew install direnv` | For per-worktree env |

Open source only. No Synopsys/Cadence/Mentor anywhere.

### 2.11 Rollout — phased, each phase reversible

Six phases, each ending in a still-working Emacs:

| # | Phase | Risk | Reversible? |
|---|-------|------|-------------|
| 0 | Pre-flight: tarball backup + `git init` in doom dir | Zero | N/A |
| 1 | Doom module + package additions | Low | `git revert` |
| 2 | SystemVerilog tooling (Verible LSP + verilog-ts-mode + verilog-ext) | Low | `git revert` + `brew uninstall` |
| 3 | Dynamic daemon orchestration (`eda-daemons.el` + `eda` CLI) | Medium | `rm` and unload |
| 4 | Per-task `project.org` + org-agenda auto-discovery | Low | `git revert` |
| 5 | Compile/sim/wave/formal integration (`SPC c *`) | Medium | `git revert` + uninstall brew |
| 6 | Claude workflow polish (`CLAUDE.md`, `.claude/agents/`, gptel) | Low | `git revert` |

You can stop at any phase boundary and have a fully working Emacs.

### 2.12 Backup — tarball + git, both belt and suspenders

**Question.** What if I break my Emacs setup?

**Decision.**

- **Before phase 0**: a dated tarball of `~/.config/doom/` and `~/.config/emacs/` to `~/backups/doom-pre-eda-<date>.tar.gz`.
- **During**: `git init` inside `~/.config/doom/`. Every phase ends with a commit. Per-change rollback via `git revert`.

Tarball survives `rm -rf .git`; git history survives a partial revert chain. Both.

### 2.13 Files location clarifier — Mac local, not Dropbox, not iCloud

- **Worktrees**: `~/eda/wt/<task>/` — Mac local filesystem, fast APFS. Not Dropbox, not iCloud, not Time Machine excluded.
- **Bare/main repos**: `~/eda/<repo>/` — Mac local. Origin is your internal Gitea.
- **Daemon registry**: `~/.config/doom/eda-daemons.el` — version-controlled (git inside doom dir).
- **Per-daemon persp state**: `~/.config/doom/.persp-state-<name>.el` — `.gitignore`d.
- **Org roots**:
  - Life: `~/Dropbox/organist-dinesh/life.org` (unchanged)
  - Per-task: `~/eda/wt/<task>/project.org` (local)
  - Cross-IP roam: `~/Dropbox/org-roam/<ip>/` (unchanged base dir, new subdirs)

### 6.8 `~/.config/doom/eda-sv.el` — SystemVerilog wiring

Create this file. Then `(load! "eda-sv")` from the end of `config.el` (last line of phase 2).

```elisp
;;; ~/.config/doom/eda-sv.el  -*- lexical-binding: t; -*-
;;;
;;; SystemVerilog stack: tree-sitter major mode, Verible LSP, verilog-ext.
;;; Loaded by config.el via (load! "eda-sv").

;; --- 1. Major-mode routing ---------------------------------------------------
;; Open all .sv/.svh/.v/.vh in verilog-ts-mode (falls back to verilog-mode
;; gracefully if the grammar is missing).
(when (treesit-available-p)
  (add-to-list 'auto-mode-alist '("\\.s?vh?\\'" . verilog-ts-mode))
  ;; .sva (assertions), .vams (verilog-AMS), .lst (filelists) — routed elsewhere
  (add-to-list 'auto-mode-alist '("\\.sva\\'" . verilog-ts-mode)))

;; --- 2. verilog-ext --------------------------------------------------------
(use-package! verilog-ext
  :after verilog-mode
  :hook ((verilog-mode    . verilog-ext-mode)
         (verilog-ts-mode . verilog-ext-mode))
  :init
  (setq verilog-ext-feature-list
        '(font-lock
          xref
          capf
          hierarchy
          eglot                 ; we'll opt into eglot; switch to lsp if preferred
          flycheck
          beautify
          navigation
          template
          formatter
          compilation
          imenu
          which-func
          hideshow
          typedefs
          time-stamp
          block-end-comments
          ports))
  :config
  (verilog-ext-mode-setup)
  ;; Tell verilog-ext to wire Verible as the eglot server
  (verilog-ext-eglot-set-server 've-verible-ls))

;; --- 3. Indentation defaults (Verible default is 2; many shops use 4) -------
(setq verilog-indent-level 2
      verilog-indent-level-module 2
      verilog-indent-level-declaration 2
      verilog-indent-level-behavioral 2
      verilog-indent-level-directive 0
      verilog-auto-newline nil
      verilog-tab-always-indent t
      verilog-case-fold nil
      verilog-highlight-modules t
      verilog-highlight-includes t
      verilog-auto-lineup 'declarations)

;; --- 4. Verible formatter (apheleia-compatible) -----------------------------
;; verilog-ext exposes verilog-ext-beautify-* commands; we also surface
;; verible-verilog-format directly for "format buffer" pipelines.
(defun eda/verible-format-buffer ()
  "Format current buffer with verible-verilog-format in place."
  (interactive)
  (when (and buffer-file-name
             (executable-find "verible-verilog-format"))
    (let ((tmp (make-temp-file "verible-fmt" nil ".sv")))
      (unwind-protect
          (progn
            (write-region (point-min) (point-max) tmp nil 'silent)
            (call-process "verible-verilog-format" nil nil nil
                          "--inplace" tmp)
            (let ((coding buffer-file-coding-system))
              (erase-buffer)
              (insert-file-contents tmp)))
        (delete-file tmp)))))

;; Opt-in: hook for format-on-save. Off by default; turn on per project via
;; `((verilog-ts-mode . ((eval . (add-hook 'before-save-hook #'eda/verible-format-buffer nil t)))))`
;; in a .dir-locals.el at the project root.

;; --- 5. compilation-error-regexp-alist additions ---------------------------
;; So that errors from verilator/yosys/verible/iverilog become clickable in
;; *compilation* buffers. Patterns are deliberately permissive — false matches
;; are cheap, missed jumps are costly.
(with-eval-after-load 'compile
  ;; Verilator: "%Error: file.sv:LINE:COL: message"
  (add-to-list 'compilation-error-regexp-alist 'verilator)
  (add-to-list 'compilation-error-regexp-alist-alist
               '(verilator
                 "^%\\(Error\\|Warning\\)[^:]*: \\([^ \n]+\\):\\([0-9]+\\):\\([0-9]+\\):"
                 2 3 4))
  ;; Verible lint: "file.sv:LINE:COL: message [rule]"
  (add-to-list 'compilation-error-regexp-alist 'verible)
  (add-to-list 'compilation-error-regexp-alist-alist
               '(verible
                 "^\\(/[^:\n]+\\.s?vh?\\):\\([0-9]+\\):\\([0-9]+\\): "
                 1 2 3))
  ;; Yosys: "ERROR: ... at file.sv:LINE"
  (add-to-list 'compilation-error-regexp-alist 'yosys)
  (add-to-list 'compilation-error-regexp-alist-alist
               '(yosys
                 "ERROR:.* at \\([^ \n]+\\.s?vh?\\):\\([0-9]+\\)" 1 2))
  ;; Icarus Verilog: "file.sv:LINE: error: message"
  (add-to-list 'compilation-error-regexp-alist 'iverilog)
  (add-to-list 'compilation-error-regexp-alist-alist
               '(iverilog
                 "^\\([^:\n]+\\.s?vh?\\):\\([0-9]+\\): \\(error\\|warning\\):"
                 1 2)))

;; --- 6. magit worktrees first-class --------------------------------------
(with-eval-after-load 'magit
  (add-hook 'magit-status-sections-hook #'magit-insert-worktrees t))

(provide 'eda-sv)
```

## 7. Phase 3 — Dynamic daemon orchestration

Goal: make every IP family a long-lived `emacs --bg-daemon=<name>` process; make `SPC k d n` spawn a new one without restarting; make `eda` from the shell drive the whole thing.

### 7.1 `~/.config/doom/eda-daemons.el` — full source

```elisp
;;; ~/.config/doom/eda-daemons.el  -*- lexical-binding: t; -*-
;;;
;;; Dynamic Emacs-daemon orchestration for the EDA IDE workflow.
;;;
;;; Concepts:
;;;   * Each IP family (soc, pcie, ucie, ...) runs as one named daemon:
;;;       emacs --bg-daemon=<name>
;;;   * A persistent registry (`eda/daemons-alist`) tracks the friendly name
;;;     and default working root for each daemon.
;;;   * The registry is loaded from disk on startup and saved on every change.
;;;   * Per-daemon persp-mode state lives at `~/.config/doom/.persp-state-<name>.el`.
;;;
;;; User-facing commands (bound under `SPC k d`):
;;;   eda/new-daemon       (SPC k d n)  — interactively create + spawn a daemon
;;;   eda/list-daemons     (SPC k d l)  — tabulated view of running daemons
;;;   eda/switch-daemon    (SPC k d s)  — `emacsclient -s <name> -c` from inside Emacs
;;;   eda/kill-daemon      (SPC k d k)  — kill by name (asks twice)
;;;   eda/rename-daemon    (SPC k d r)  — rename in registry (cosmetic only)
;;;   eda/restart-daemon   (SPC k d R)  — kill + respawn, restore persp state
;;;   eda/seed-daemons     (SPC k d S)  — first-run helper: create soc/pcie/ucie

(require 'cl-lib)
(require 'persp-mode nil 'noerror)
(require 'json)

;; --- Registry --------------------------------------------------------------

(defvar eda/registry-file
  (expand-file-name "eda-registry.el" doom-user-dir)
  "Where the daemon registry is persisted (machine-local).")

(defvar eda/daemons-alist nil
  "Registry of EDA daemons.
Each entry: (NAME . PLIST) where PLIST has keys:
  :root      -> default working directory for the daemon
  :ip-family -> short symbol, e.g. 'pcie
  :created   -> ISO timestamp string
  :notes     -> free-form string.")

(defun eda/registry-load ()
  "Load the daemon registry from disk."
  (when (file-readable-p eda/registry-file)
    (with-temp-buffer
      (insert-file-contents eda/registry-file)
      (setq eda/daemons-alist (read (current-buffer))))))

(defun eda/registry-save ()
  "Persist the registry to disk."
  (with-temp-file eda/registry-file
    (let ((print-level nil) (print-length nil))
      (insert ";; eda-registry.el — auto-generated, edit with care.\n")
      (prin1 eda/daemons-alist (current-buffer)))))

(eda/registry-load)

;; --- persp-mode state per daemon -------------------------------------------

(defun eda/persp-state-file (&optional name)
  "Return path to the persp-state file for the current (or NAME) daemon."
  (expand-file-name
   (format ".persp-state-%s.el"
           (or name (daemonp) "default"))
   doom-user-dir))

(defun eda/persp-save-on-shutdown ()
  "Save persp-mode workspaces on Emacs shutdown, namespaced by daemon."
  (when (and (featurep 'persp-mode) (daemonp))
    (let ((persp-auto-save-fname (file-name-nondirectory (eda/persp-state-file)))
          (persp-save-dir        (file-name-directory   (eda/persp-state-file))))
      (ignore-errors (persp-save-state-to-file persp-auto-save-fname)))))

(defun eda/persp-load-on-startup ()
  "Load this daemon's persp-mode state at startup."
  (when (and (featurep 'persp-mode) (daemonp)
             (file-readable-p (eda/persp-state-file)))
    (ignore-errors
      (persp-load-state-from-file (eda/persp-state-file)))))

(add-hook 'kill-emacs-hook       #'eda/persp-save-on-shutdown)
(add-hook 'emacs-startup-hook    #'eda/persp-load-on-startup)

;; --- Probing running daemons (system view, not registry) -------------------

(defun eda/running-daemons ()
  "Return a list of daemon names currently running on this machine."
  (let ((out (shell-command-to-string
              "pgrep -fa 'emacs.*--\\(bg-\\)?daemon=' 2>/dev/null")))
    (cl-loop for line in (split-string out "\n" t)
             when (string-match "--\\(?:bg-\\)?daemon=\\([^ \t]+\\)" line)
             collect (match-string 1 line))))

(defun eda/daemon-running-p (name)
  "Return non-nil if a daemon named NAME is alive."
  (member name (eda/running-daemons)))

;; --- Spawning a new daemon --------------------------------------------------

(defun eda/--validate-name (name)
  "Sanity-check NAME (alnum + dashes only, 1-32 chars)."
  (unless (string-match-p "\\`[a-z0-9][a-z0-9-]\\{0,31\\}\\'" name)
    (user-error "Daemon name must match [a-z0-9][a-z0-9-]{0,31}: %s" name)))

(defun eda/spawn-daemon (name root)
  "Spawn `emacs --bg-daemon=NAME` with default-directory ROOT."
  (eda/--validate-name name)
  (when (eda/daemon-running-p name)
    (user-error "Daemon %s is already running" name))
  (let ((default-directory (file-name-as-directory root)))
    (start-process (format "eda-spawn-%s" name) nil
                   (executable-find "emacs")
                   (format "--bg-daemon=%s" name))))

;;;###autoload
(defun eda/new-daemon (name root &optional ip-family notes)
  "Interactively create and spawn a new EDA daemon.
NAME is a short token; ROOT is its default working directory."
  (interactive
   (let* ((n (read-string "Daemon name (e.g. cxl, hbm-phy): "))
          (r (read-directory-name (format "Default root for %s: " n)
                                  "~/eda/wt/"))
          (f (intern (read-string "IP family symbol (e.g. cxl): " n)))
          (notes (read-string "Notes (optional): ")))
     (list n r f notes)))
  (eda/--validate-name name)
  (when (assoc name eda/daemons-alist)
    (user-error "%s already in registry; pick another name" name))
  (push (cons name (list :root      (expand-file-name root)
                         :ip-family ip-family
                         :created   (format-time-string "%Y-%m-%dT%H:%M:%S")
                         :notes     notes))
        eda/daemons-alist)
  (eda/registry-save)
  (eda/spawn-daemon name root)
  (message "Spawned daemon %s @ %s" name root))

;;;###autoload
(defun eda/seed-daemons ()
  "First-run helper: create the initial soc / pcie / ucie daemons."
  (interactive)
  (dolist (spec '(("soc"  "~/eda/wt/" soc  "SoC integration top-level")
                  ("pcie" "~/eda/wt/" pcie "PCIe Gen7 controller + PHY")
                  ("ucie" "~/eda/wt/" ucie "UCIe sideband + retimer + D2D")))
    (cl-destructuring-bind (name root family notes) spec
      (unless (assoc name eda/daemons-alist)
        (push (cons name (list :root      (expand-file-name root)
                               :ip-family family
                               :created   (format-time-string "%Y-%m-%dT%H:%M:%S")
                               :notes     notes))
              eda/daemons-alist))
      (unless (eda/daemon-running-p name)
        (eda/spawn-daemon name root))))
  (eda/registry-save)
  (message "Seeded daemons: soc, pcie, ucie"))

;; --- Listing ---------------------------------------------------------------

(define-derived-mode eda/daemons-mode tabulated-list-mode "EDA-Daemons"
  "Tabulated view of EDA daemons."
  (setq tabulated-list-format
        [("Name"     12 t)
         ("Running?"  8 t)
         ("Family"   10 t)
         ("Root"     40 t)
         ("Notes"    30 nil)])
  (tabulated-list-init-header))

;;;###autoload
(defun eda/list-daemons ()
  "Pop up a buffer listing all EDA daemons + their status."
  (interactive)
  (let ((buf (get-buffer-create "*EDA Daemons*"))
        (running (eda/running-daemons)))
    (with-current-buffer buf
      (eda/daemons-mode)
      (setq tabulated-list-entries
            (cl-loop for (name . plist) in eda/daemons-alist
                     collect
                     (list name
                           (vector name
                                   (if (member name running) "yes" "no")
                                   (format "%s" (or (plist-get plist :ip-family) ""))
                                   (or (plist-get plist :root) "")
                                   (or (plist-get plist :notes) "")))))
      (tabulated-list-print)
      (pop-to-buffer buf))))

;; --- Switching (attach) ----------------------------------------------------

;;;###autoload
(defun eda/switch-daemon (name)
  "Attach a new GUI frame to daemon NAME (or spawn it if dead)."
  (interactive
   (list (completing-read "Daemon: "
                          (mapcar #'car eda/daemons-alist) nil t)))
  (unless (eda/daemon-running-p name)
    (when-let ((root (plist-get (cdr (assoc name eda/daemons-alist)) :root)))
      (eda/spawn-daemon name root)
      (sleep-for 1)))
  (start-process (format "eda-attach-%s" name) nil
                 (executable-find "emacsclient")
                 "-s" name "-c" "-n"))

;; --- Kill ------------------------------------------------------------------

;;;###autoload
(defun eda/kill-daemon (name)
  "Kill daemon NAME. Asks twice to avoid accidents."
  (interactive
   (list (completing-read "Kill daemon: "
                          (eda/running-daemons) nil t)))
  (when (yes-or-no-p (format "Really kill daemon %s? " name))
    (when (yes-or-no-p (format "REALLY really? This may lose unsaved buffers in %s. " name))
      (call-process (executable-find "emacsclient") nil nil nil
                    "-s" name "-e" "(kill-emacs)")
      (message "Sent kill to %s" name))))

;;;###autoload
(defun eda/rename-daemon (old new)
  "Rename a daemon in the registry. Does NOT touch the running process.
Use only between sessions of OLD, or run eda/restart-daemon after."
  (interactive
   (let ((o (completing-read "Old name: " (mapcar #'car eda/daemons-alist) nil t)))
     (list o (read-string (format "Rename %s to: " o)))))
  (eda/--validate-name new)
  (when-let ((cell (assoc old eda/daemons-alist)))
    (setf (car cell) new)
    (eda/registry-save))
  (message "Renamed %s -> %s (registry only)" old new))

;;;###autoload
(defun eda/restart-daemon (name)
  "Kill + respawn daemon NAME, preserving its registry entry + persp state."
  (interactive
   (list (completing-read "Restart: " (mapcar #'car eda/daemons-alist) nil t)))
  (when (eda/daemon-running-p name)
    (call-process (executable-find "emacsclient") nil nil nil
                  "-s" name "-e" "(kill-emacs)")
    (sleep-for 2))
  (when-let ((root (plist-get (cdr (assoc name eda/daemons-alist)) :root)))
    (eda/spawn-daemon name root)
    (message "Restarted %s" name)))

;; --- Worktree pairing -----------------------------------------------------

;;;###autoload
(defun eda/new-worktree-for-task (repo branch task)
  "Convenience: create a worktree at ~/eda/wt/<task>/ off REPO/BRANCH."
  (interactive
   (let* ((r (read-directory-name "Repo (main checkout): " "~/eda/"))
          (b (read-string "Branch / ref: " "main"))

                  "-C" repo "worktree" "add" "-b" task wt branch)
    (message "Worktree ready at %s" wt)
    (find-file (expand-file-name "." wt))))
```

```bash
# Companion to ~/.config/doom/eda-daemons.el.
#
# Usage:
#   eda ls                   list all running daemons
#   eda new <name> [path]    create + spawn a new daemon
#   eda <name>               attach a GUI frame to <name> (spawn if dead)
#   eda kill <name>          stop daemon <name>
#   eda restart <name>       restart daemon <name>
#   eda exec <name> <elisp>  evaluate elisp in <name>
#   eda paths                print canonical paths used by this tool
#
# Conventions:
#   * Daemons are run as `emacs --bg-daemon=<name>`.
#   * Registry is in ~/.config/doom/eda-registry.el (managed from Emacs).
#   * Logs in ~/.cache/eda/<name>.log if explicitly logged via this wrapper.

set -euo pipefail

EMACS="${EMACS:-/opt/homebrew/bin/emacs}"
EMACSCLIENT="${EMACSCLIENT:-/opt/homebrew/bin/emacsclient}"
LOG_DIR="${HOME}/.cache/eda"
WT_ROOT="${HOME}/eda/wt"
REGISTRY="${HOME}/.config/doom/eda-registry.el"

mkdir -p "${LOG_DIR}" "${WT_ROOT}"

die() { echo "eda: $*" >&2; exit 1; }

usage() {
  sed -n '2,/^set -euo pipefail/p' "$0" | sed 's/^# \{0,1\}//'
  exit 0
}

cmd_ls() {
  printf "%-16s %-8s %-10s %s\n" NAME RUNNING PID ROOT
  printf "%s\n" "------------------------------------------------------------"
  while read -r line; do
    [[ -z "$line" ]] && continue
    pid=$(echo "$line" | awk '{print $1}')
    name=$(echo "$line" | sed -nE 's/.*--(bg-)?daemon=([^ ]+).*/\2/p')
    [[ -z "$name" ]] && continue
    # Best-effort root lookup from registry
    root=$(grep -oE "\"$name\"[^\"]*\"[^\"]+\"" "$REGISTRY" 2>/dev/null | head -1 | awk -F\" '{print $4}')
    printf "%-16s %-8s %-10s %s\n" "$name" yes "$pid" "${root:-?}"
  done < <(pgrep -fa 'emacs.*--(bg-)?daemon=' || true)
}

cmd_new() {
  local name="${1:-}" path="${2:-${WT_ROOT}}"
  [[ -z "$name" ]] && die "usage: eda new <name> [path]"
  [[ "$name" =~ ^[a-z0-9][a-z0-9-]{0,31}$ ]] || die "bad name: $name"
  if pgrep -fa "emacs.*--(bg-)?daemon=${name}\b" >/dev/null; then
    die "daemon ${name} already running"
  fi
  mkdir -p "$path"
  ( cd "$path" && nohup "$EMACS" --bg-daemon="$name" >"${LOG_DIR}/${name}.log" 2>&1 & )
  # Register in Emacs registry by sending elisp to any running daemon
  local target
  target=$(pgrep -fa 'emacs.*--(bg-)?daemon=' | head -1 | sed -nE 's/.*--(bg-)?daemon=([^ ]+).*/\2/p' || true)
  if [[ -n "${target:-}" ]]; then
    "$EMACSCLIENT" -s "$target" -e \
      "(progn (require 'eda-daemons) (unless (assoc \"$name\" eda/daemons-alist) (push (cons \"$name\" (list :root (expand-file-name \"$path\") :ip-family (intern \"$name\") :created (format-time-string \"%Y-%m-%dT%H:%M:%S\") :notes \"\")) eda/daemons-alist) (eda/registry-save)))" \
      >/dev/null || true
  fi
  echo "Spawned daemon ${name} at ${path}"
}

cmd_attach() {
  local name="$1"
  if ! pgrep -fa "emacs.*--(bg-)?daemon=${name}\b" >/dev/null; then
    # Try to spawn from registry root
    local root
    root=$(grep -oE "\"$name\"[^\"]*\"[^\"]+\"" "$REGISTRY" 2>/dev/null | head -1 | awk -F\" '{print $4}')
    cmd_new "$name" "${root:-${WT_ROOT}}"
    sleep 1
  fi
  "$EMACSCLIENT" -s "$name" -c -n
}

cmd_kill() {
  local name="$1"
  pgrep -fa "emacs.*--(bg-)?daemon=${name}\b" >/dev/null || die "no daemon $name"
  "$EMACSCLIENT" -s "$name" -e '(kill-emacs)' >/dev/null
  echo "killed $name"
}

cmd_restart() {
  cmd_kill "$1"
  sleep 1
  cmd_new "$1"
}

cmd_exec() {
  local name="$1"; shift
  "$EMACSCLIENT" -s "$name" -e "$*"
}

cmd_paths() {
  cat <<EOF
EMACS=$EMACS
EMACSCLIENT=$EMACSCLIENT
LOG_DIR=$LOG_DIR
WT_ROOT=$WT_ROOT
REGISTRY=$REGISTRY
EOF
}

main() {
  local sub="${1:-}"
  case "$sub" in
    ""|-h|--help|help) usage ;;
    ls)        shift; cmd_ls "$@" ;;
    new)       shift; cmd_new "$@" ;;
    kill)      shift; cmd_kill "$@" ;;
    restart)   shift; cmd_restart "$@" ;;
    exec)      shift; cmd_exec "$@" ;;
    paths)     cmd_paths ;;
    *)         cmd_attach "$sub" ;;     # `eda pcie` -> attach pcie
  esac
}

main "$@"
```

## 8. Phase 4 — Per-task `project.org` + agenda auto-discovery

Goal: every worktree carries its own `project.org`; opening any of them surfaces TODO/agenda for that task; new tasks appear in `org-agenda-files` without manual edits. The existing `life.org` Eisenhower flow remains untouched.

### 8.1 `~/.config/doom/eda-tasks.el` — full source

```elisp
;;; ~/.config/doom/eda-tasks.el  -*- lexical-binding: t; -*-
;;;
;;; Per-task project.org + org-agenda auto-discovery.

(require 'org)
(require 'org-capture)

(defvar eda/worktree-root (expand-file-name "~/eda/wt/")
  "Where all per-task worktrees live.")

(defvar eda/project-org-filename "project.org"
  "Name of the per-task org file inside each worktree.")

;; --- 1. agenda auto-discovery ----------------------------------------------

(defun eda/all-project-org-files ()
  "Return every project.org under eda/worktree-root (depth 2)."
  (when (file-directory-p eda/worktree-root)
    (let (acc)
      (dolist (wt (directory-files eda/worktree-root t "\\`[^.]"))
        (when (file-directory-p wt)
          (let ((f (expand-file-name eda/project-org-filename wt)))
            (when (file-readable-p f) (push f acc)))))
      acc)))

(defun eda/refresh-agenda-files ()
  "Re-scan worktrees + append project.org files to org-agenda-files.
Preserves any non-EDA files already in the list (e.g. life.org)."
  (interactive)
  (let* ((existing (cl-remove-if
                    (lambda (f) (string-prefix-p eda/worktree-root f))
                    (or org-agenda-files '())))
         (eda     (eda/all-project-org-files))
         (life    (list (expand-file-name "life.org" (or org-directory "~/")))))
    (setq org-agenda-files (delete-dups (append life existing eda)))))

;; Refresh on startup and whenever we save a project.org
(add-hook 'emacs-startup-hook #'eda/refresh-agenda-files)
(add-hook 'after-save-hook
          (lambda ()
            (when (and buffer-file-name
                       (string= (file-name-nondirectory buffer-file-name)
                                eda/project-org-filename))
              (eda/refresh-agenda-files))))

;; --- 2. project.org template (created on demand) ---------------------------

(defun eda/project-org-template (task)
  "Return string contents for a fresh project.org for TASK."
  (format
   "#+TITLE: %s
#+AUTHOR: Dinesh Bhardwaj
#+DATE: %s
#+CATEGORY: %s
#+FILETAGS: :eda:%s:
#+STARTUP: overview
#+TODO: TODO IN-PROGRESS BLOCKED REVIEW | DONE WONTDO

* Goal
  - One-line statement of what \"done\" means for this task.

* Context links
  - RTL roots: [[file:rtl/]]
  - Testbench: [[file:tb/]]
  - Sim build:  [[file:sim/]]
  - Spec / IP-XACT: TODO

* TODOs
** TODO Investigate / scoping
** TODO RTL implementation
** TODO Verification (UVM / cocotb / formal)
** TODO Coverage closure
** TODO Lint clean
** TODO PR + Forge review

* Debug log
  Append-only; timestamped entries.

* Decisions
  Capture the why, not the what. Cross-link to org-roam nodes.

* Open questions
  Things blocking forward progress; mention the human / link to chat.
"
   task
   (format-time-string "%Y-%m-%d")
   task task))

(defun eda/ensure-project-org (&optional dir)
  "Create project.org in DIR (default: current project root) if missing."
  (interactive)
  (let* ((root (or dir
                   (projectile-project-root)
                   default-directory))
         (file (expand-file-name eda/project-org-filename root))
         (task (file-name-nondirectory (directory-file-name root))))
    (unless (file-exists-p file)
      (with-temp-file file
        (insert (eda/project-org-template task)))
      (message "Wrote %s" file))
    (find-file file)
    (eda/refresh-agenda-files)))

;; --- 3. New capture templates (NEW keys p* — do not collide with i/u/d/w/x)
;; Existing keys in config.el: i, u, d, w, x (all under top-level).
;; We use a new top-level key "p" for project-task capture so no collision.

(with-eval-after-load 'org
  (add-to-list 'org-capture-templates
               '("p" "Project task captures (per-worktree project.org)"))
  (add-to-list 'org-capture-templates
               `("pt" "Project TODO" entry
                 (function eda/-capture-target-todos)
                 "* TODO %?  %^g\n  CREATED: %U\n  %a\n"
                 :empty-lines 1))
  (add-to-list 'org-capture-templates
               `("pd" "Project debug log" entry
                 (function eda/-capture-target-debug)
                 "* %U  %?\n  %a\n"
                 :empty-lines 1))
  (add-to-list 'org-capture-templates
               `("pq" "Project quick note" item
                 (function eda/-capture-target-questions)
                 "- %?  %U\n"
                 :empty-lines 0)))

(defun eda/-current-project-org ()
  "Find the project.org for the current buffer/project."
  (let ((root (or (and (projectile-project-p) (projectile-project-root))
                  default-directory)))
    (expand-file-name eda/project-org-filename root)))

(defun eda/-capture-target-todos ()
  (find-file (eda/-current-project-org))
  (goto-char (point-min))
  (re-search-forward "^\\* TODOs" nil t))

(defun eda/-capture-target-debug ()
  (find-file (eda/-current-project-org))
  (goto-char (point-min))
  (re-search-forward "^\\* Debug log" nil t))

(defun eda/-capture-target-questions ()
  (find-file (eda/-current-project-org))
  (goto-char (point-min))
  (re-search-forward "^\\* Open questions" nil t))

;; --- 4. Keybindings under SPC p (project) — uses Doom default prefix -------
;; Doom already binds SPC p as project (projectile). We add EDA-specific
;; commands that do not collide.

(map! :leader
      (:prefix-map ("p e" . "eda project")
       :desc "Open project.org"           "o" #'eda/ensure-project-org
       :desc "Refresh agenda from wt/"    "r" #'eda/refresh-agenda-files
       :desc "Capture TODO -> project"    "t" (lambda () (interactive) (org-capture nil "pt"))
       :desc "Capture debug -> project"   "d" (lambda () (interactive) (org-capture nil "pd"))
       :desc "Capture question -> project" "q" (lambda () (interactive) (org-capture nil "pq"))))

(provide 'eda-tasks)
```

### 10.4 Multi-daemon Claude pattern

Pattern for keeping each IP family's Claude sessions isolated:

1. Each daemon (`soc`, `pcie`, `ucie`, …) has its **own** vterm running `claude` rooted at `~/eda/wt/<task>/`.

3. The session's `CLAUDE.md` is the per-worktree file we seeded — Claude reads it on every conversation start.
4. To share design decisions across IPs without polluting context: keep cross-IP knowledge in org-roam, not in CLAUDE.md. Reference by URL when needed.

## 14. Open questions / followups

Items where the plan could not fully resolve a tradeoff; resolve with user input before committing the related phase.

1. **lsp-mode vs eglot.** Phase 1 enables `:tools lsp` with the default (lsp-mode). verilog-ext supports both. lsp-mode has richer UI (peek, sideline) and Doom's `:tools lookup` integrates more tightly with it; eglot is lighter, built-in (Emacs 29+), and better at TRAMP. **Question for Dinesh.** Default to lsp-mode (current plan) or switch to eglot (`+eglot` flag)? Recommendation: start with lsp-mode; switch only if startup feels heavy.

2. **OpenLane2 / OpenROAD on Apple Silicon.** Both work only through Docker on Mac, and the Mac-on-arm Docker performance is acceptable for small designs but noticeable for full SoC P&R. **Question.** Is GDS-II flow in scope for this IDE, or do you offload P&R to a Linux farm via git push? Plan currently treats P&R as out-of-scope on the Mac (only RTL + verification + formal locally).

3. **PCIe Gen7 simulator coverage.** Verilator 5.x handles the controller RTL fine, but PHY-level (PIPE, 64b/66b, ML5/6) wants UVM + real PCIe BFMs. Open-source PCIe BFMs are limited (mainly Cocotb-PCIe-Express + community gen3/4 models). **Question.** For Gen7 PHY work, do you accept a hybrid model where PHY-loopback uses a SystemVerilog harness and PCIe BFMs are mocked in Python via cocotb? Or do you keep PHY on a commercial sim that lives outside this IDE? Plan assumes the former.

4. **Daemon attach UX on first run.** `eda <name>` will spawn the daemon if not running, but the first call has a ~1-2s race window where `emacsclient -c` may attach before the daemon is fully ready. Plan uses `sleep 1`. **Question.** Acceptable, or do you want a retry-with-backoff loop? Recommendation: keep `sleep 1`; revisit only if you hit it.

5. **persp-mode state file format.** Doom's `:ui workspaces` uses persp-mode but Doom wraps persistence in `+workspace/save` and `+workspace/load`. Per-daemon namespacing via `persp-auto-save-fname` should work but is not heavily tested across Doom's workspace wrapper. **Question.** If the daemon's workspaces fail to restore on restart, do you want to fall back to Doom's `+workspace/save` invoked from a `kill-emacs-hook`? Plan ships the persp-mode-direct approach first.

6. **Forge with worktrees.** Forge has known papercuts where issue/PR data is keyed by repo URL — multiple worktrees share that DB which is the right behaviour but Forge sometimes confuses local checkouts. **Question.** Is your Gitea Forge integration already mature enough that this doesn't matter? Plan leaves it untouched.

7. **Verible vs slang as primary lint.** Verible is the LSP; for **heavy** lint (Gen7 controller scale) slang is faster and more compliant. **Question.** Do you want me to wire slang as a secondary flycheck checker (in addition to Verible LSP diagnostics) in phase 2, or save it for phase 7? Plan defers slang to a future phase.

## 15. Rollback procedures

### 15.1 Per-phase rollback

Every phase ends with a git commit in `~/.config/doom`. Rollback is `git revert <SHA>` plus the targeted brew/file cleanup listed at the end of each phase.

```bash
# Quick reference
cd ~/.config/doom
git log --oneline | head -10               # find the phase SHA
git revert <SHA>                           # creates a revert commit
doom sync                                  # if packages.el changed
# Then run any uninstall steps from the phase's rollback section.
```

### 15.2 Full rollback (nuke and pave)

```bash
# 1. Stop all EDA daemons
for d in $(pgrep -fa 'emacs.*--bg-daemon=' | sed -nE 's/.*--bg-daemon=([^ ]+).*/\1/p'); do
  emacsclient -s "$d" -e '(kill-emacs)' 2>/dev/null || true
done

# 2. Remove EDA-specific files
rm -f ~/.config/doom/eda-{daemons,sv,sim,tasks,claude}.el
rm -f ~/.config/doom/eda-registry.el
rm -f ~/.config/doom/.persp-state-*.el
rm -rf ~/.config/doom/CLAUDE-templates ~/.config/doom/agent-templates
rm -f ~/.local/bin/eda
rm -f ~/Library/LaunchAgents/io.cspeed.eda-*.plist

# 3. Restore Doom config to pre-EDA snapshot
cd ~/.config/doom
rm -rf .git              # only if you also want to discard the git history
tar -xzf ~/backups/doom-pre-eda-*.tar.gz -C ~/.config/

# 4. Uninstall EDA brew formulae if desired (optional — they don't bother Doom)
brew uninstall verible verilator icarus-verilog yosys graphviz direnv
brew uninstall --cask gtkwave
brew untap chipsalliance/verible randomplum/gtkwave

# 5. Resync Doom
doom sync
doom doctor
```

### 15.3 Sanity check after rollback

After a full rollback you should observe:

- `SPC o R` opens elfeed-summary (existing).
- `SPC k k` starts Claude Code (existing).
- `SPC m s` summarises an elfeed entry (existing).
- `SPC g f` runs forge-dispatch (existing).
- Org-capture `i / u / d / w / x` write to `life.org` (existing).
- No `SPC k d` / `SPC k a` / `SPC c e` / `SPC p e` prefixes exist.

If any existing behaviour is broken after rollback: extract the tarball into `/tmp/doom-good`, `diff -r /tmp/doom-good ~/.config/doom`, and re-apply by hand.
