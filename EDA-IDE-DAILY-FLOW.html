01 — The mental model

Four ideas the whole system rests on

A

The org heading is the task

One heading = one worktree = one perspective = one Claude session. You act on the heading; everything else follows.

B

One command to get anywhere

SPC k o j switches to the task's worktree and raises/resumes its Claude session. Your single point of contact.

C

Effort clock ≠ org clock

Clock several tasks at once — each earns full time. Idle overlap is subtracted only at report time; raw CLOCK: lines are never touched.

D

Sessions are deterministic

Each task stores its own :CLAUDE_SESSION: id, so Claude is always reattached by that stamped id — --resume when a transcript exists, else created under the same id with --session-id — never scraped from log files.

02 — Lifecycle

A task moves through four states

→

Full keyword set (Doom stock + our REVIEW):

TODO · PROJ · LOOP · STRT · WAIT · HOLD · IDEA · REVIEW  |  DONE · KILL

STRT autostarts the session for EDA tasks. REVIEW keeps the session alive so SPC k o j lands you on running Claude for the review chat. DONE is only reachable through the gate; a veto drops you back to REVIEW.

04 — A day in the life

From morning orient to the weekly report

☀ Morning

Orient

Start Emacs (or emacsclient -nw). The grid comes up with your agenda in slot 0 — skim it and pick your work.

Optional health check: SPC k o P (doctor) confirms profile, roots, tools, and which features are live on this machine.

▶ Start a task

Init → STRT → clock in

First time on a heading? Stamp the schema: SPC k o i. Each prompt autofills a default (current value, else your last-used) and completes over your history + the worktrees on disk — RET takes the default, TAB picks a past value. In a hurry, SPC k o I asks only for the slug. If the worktree dir doesn't exist yet, init offers to create it — a git worktree (repo + branch) or a plain directory; do it any time with SPC k o w. Then mark it STRT — Claude auto-starts.

Clock in for time tracking: SPC k o c — starts the timer, ensures the session, and rebuilds the default grid so this task gets its own Claude window (fires even if you'd zoomed a pane with C-x 1). SPC p f now finds files scoped to that worktree.

🌳 Worktree mode

Spin up task dirs the cheap way

Prefer a git worktree over a full git clone for each task dir. Both give an isolated dir under ~/eda/wt/<slug>/ with its own branch, index/HEAD, and SPC p f scope — but a worktree shares the one repo's .git object store, so it's near-instant, costs almost no disk, and one fetch reaches every worktree. A clone re-copies the whole history each time.

Create via SPC k o i / SPC k o w → pick git worktree, then repo + branch. One task = one branch (a branch checks out in only one worktree at a time). Parallel tasks stay isolated — a separate index per worktree means concurrent commits never touch each other. Clean up a finished task with git worktree remove ~/eda/wt/<slug> then git worktree prune — don't rm the dir by hand. And don't move or delete the hub clone, or the linked worktrees break.

⇄ Parallel

Work several tasks at once

Clock in another heading with SPC k o c — overlapping clocks are allowed and each earns full time. The grid relayouts automatically.

Move around with SPC k o j (by task) or M-o / s-N (by window). SPC k o l lists what's running.

☕ Breaks

Toggle idle

Stepping away? SPC k o z starts the per-environment Idle clock. It's a toggle — press SPC k o z again when you're back to end idle. On the way out it drops a non-destructive IDLE_ADJ ▶ overlapped … note onto every clocked task, and overlapping idle time is subtracted at report time.

Idle also ends with SPC k o 0 (clock out all), or z in the agenda.

✓ Finish

REVIEW → DONE

Move the task to REVIEW; the session stays live for a final review chat via SPC k o j.

Run SPC k o d — the gate checks commit + review + Claude self-review + memory. Don't want to hand-write the memory note? SPC k o m lets Claude distill it.

🌙 End of day

Clock out

SPC k o 0 clocks out everything and kills those sessions — or SPC k o C one at a time. Anything left in STRT/REVIEW is tomorrow's starting point.

📅 End of week

The report writes itself

SPC k o r generates reports/weekly-<isoweek>.org and opens it. See section 06.

06 — The DONE gate

DONE is earned, not typed

SPC k o d runs four checks in order, advancing a :DELIVERY: marker from pending to done. Any veto resets the task to REVIEW.

1

Committed

Worktree is clean (git status porcelain). Dirty? It offers to auto-commit. Not a git repo → this check is voided (N/A).

2

Review prompt

Three quick questions — delivered? / tested? / follow-ups? — logged into the task's LOGBOOK.

3

Claude self-review

claude -p reviews your branch diff; you accept or veto. If Claude is unavailable, you can waive-with-log.

4

Memory

A non-trivial entry must exist in the task's :MEM_SCOPE: store. Missing? It offers to distill one (or writes a stub) and fails the gate so a human reviews memory first.

On all-pass: the session is killed, the transcript is committed as a git-checkable delivery, :DELIVERY: done, and the state flips to DONE.

07 — Weekly report

The numbers fall out for free

SPC k o r writes reports/weekly-<isoweek>.org for the current ISO week (Mon 00:00 onward). Everything is computed in elisp, so the numbers are exact and testable.

  • Collective time — gross, idle-overlap, and net (gross − idle∩task).
  • By tag — net time per tag (billable, project, …).
  • By client — net time per client.
  • By task — net time per task, sorted high → low.
  • Delivered digest — every task marked DONE this week, with its review line.
Non-destructive idle math: raw CLOCK: lines stay intact; overlap is merged and subtracted only in the report. v c in the agenda still audits raw overlap. Headless runs use eda/report-weekly-batch (launchd/cron, no frame).

08 — Client, mobile & memory

Two machines, two memory stores

CLIENT

Isolated Linux box

Runs its own Emacs + Claude; never SSH'd from the Mac. Bridged by email + Dropbox, not a live link. On the Mac, client tasks.org is read-only — you queue additions to an outbox with SPC k y a / SPC k y i, then sync with SPC k y e / SPC k y m. Re-import never duplicates.

MOBILE

organice + Dropbox

Over personal.org + the append-only outbox. Folder sync only — no extra tooling.

MEM · P

Personal store

~/.claude/memory/personal/ — syncs everywhere. Distilled per-DONE, or on demand with SPC k o m.

MEM · C

Client store

~/.claude/memory/clients/<x>/ — lives on the client box only, never synced off. The :MEM_SCOPE: property picks the store.
